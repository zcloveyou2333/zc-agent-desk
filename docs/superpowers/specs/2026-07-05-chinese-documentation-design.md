# 全中文文档改造设计

## 目标

将仓库中的 `README.md` 与 `docs/**/*.md` 全部改为中文，不保留英文原版，
让 HR 和中文技术面试官能够连续阅读项目背景、架构、决策、实施过程和录屏材料。

## 范围

本次覆盖当前仓库中的全部 Markdown 文档，包括：

- 根目录 README；
- 架构、设计、决策、进度、AI 协作和录屏文档；
- `docs/superpowers/specs/` 下的历史设计规格；
- `docs/superpowers/plans/` 下的历史实施计划。

不修改 Python、TypeScript、Shell、JSON 或环境配置文件。

## 翻译规则

以下内容翻译为自然、准确的中文：

- 标题、段落、表格说明、列表说明；
- 测试意图、验收条件、取舍理由和故障说明；
- Mermaid 图中的解释性节点名称；
- 面向读者的界面名称和操作说明。

以下内容保持原样：

- 代码块中的代码与命令；
- 文件路径、URL、Git commit、版本号；
- API 路径、事件名、状态值、环境变量和函数名；
- React、FastAPI、SQLite、SSE、Hermes、Workflow、Real Agent、Mock、Trace
  等项目或行业专有名词；
- 用户实际需要输入的演示提示词。

代码块中的注释只有在不会改变复制运行结果时才翻译；Shell 命令旁的操作性注释可以翻译，
程序测试样例和字符串断言保持其原有语义。

## 表达标准

- 使用简体中文和统一术语，不做逐词直译；
- `Workflow` 表示零 Key 的确定性工作流，`Real Agent` 表示 Hermes 实时模式；
- 首次出现的重要英文缩写可附中文解释，后续保持缩写；
- 历史计划仍保留历史语气、任务状态和验证证据，不改写为“已经完成”；
- 不添加未经验证的新能力或更强的安全声明。

## 实施分批

1. 翻译核心交付材料：README、进度、录屏、AI 协作；
2. 翻译架构、设计、决策和总实施计划；
3. 翻译历史规格与分阶段实施计划；
4. 扫描残留英文自然语言，区分允许保留的技术标识符和遗漏段落；
5. 验证 Markdown 链接、代码块、测试、构建和发布扫描。

## 验收标准

- 所有目标文档的标题和说明性正文均为中文；
- 不再存在整段英文自然语言；
- 命令、API、路径、标识符和代码样例没有被误改；
- 仓库内部 Markdown 链接均指向存在的文件；
- 后端测试、前端测试、前端构建与发布扫描全部通过；
- Git diff 只包含文档及为本次翻译新增的中文计划文件。
